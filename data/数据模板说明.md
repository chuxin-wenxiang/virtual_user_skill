# 场景库数据模板说明

> 用于指导如何准备 Excel 数据文件

---

## 📊 Excel 格式要求

### 文件要求

- **格式**: Excel (.xlsx)
- **Sheet 名称**: Sheet1（只读取第一个 Sheet）
- **编码**: UTF-8
- **首行**: 字段名（必须包含）

### 必填字段

| 字段名 | 类型 | 说明 | 示例 |
|:---|:---|:---|:---|
| 序号 | 数字 | 记录编号 | 1 |
| 用户姓名和背景信息 | 文本 | 用户画像描述 | 姓名 [廖慧] 所属八大人群 [新锐白领] 年龄 [28]... |
| 用户具体场景 | 文本 | 使用场景 | 行程规划和攻略制定 |
| 任务 | 文本 | 用户任务 | 为亚洲短途旅行确定合适的预订启动时间 |
| 底层需求 | 文本 | 根本需求 | 对旅行规划有掌控感和确定性 |

### 可选字段

| 字段名 | 说明 |
|:---|:---|
| 内容范围 | 内容分类（如：出境游） |
| 是否出境游 | 是否出境（是/否） |
| 期待效果 | 期望结果 |
| 当前方案 | 当前解法 |
| 爽点 | 满意点 |
| 痛点 | 不满意点 |
| 改进方向 | 改进建议 |

## 🔧 数据准备步骤

### 步骤 1：从语雀导出

1. 打开语雀文档
2. 点击"导出" → "Excel"
3. 保存为 `scenario_library.xlsx`

### 步骤 2：检查字段

确保包含以下必填字段：
- [ ] 序号
- [ ] 用户姓名和背景信息
- [ ] 用户具体场景
- [ ] 任务
- [ ] 底层需求

### 步骤 3：转换数据

```bash
python3 scripts/prepare_data.py scenario_library.xlsx
```

### 步骤 4：验证输出

检查生成的文件：
- [ ] `data/scenario_library.json.enc`（加密数据）
- [ ] `data/scenario_embeddings.npy`（向量文件）
- [ ] `data/sample_scenarios.json`（示例数据，前 5 条）

## ⚠️ 注意事项

### 数据质量

1. **字段完整性**: 必填字段不能为空
2. **数据格式**: 用户背景信息格式统一（姓名 [xxx] 所属八大人群 [xxx]...）
3. **文本长度**: 避免过长文本（建议<500 字）
4. **特殊字符**: 避免包含 JSON 特殊字符（如未转义的引号）

### 隐私保护

1. **脱敏处理**: 用户姓名可使用化名
2. **敏感信息**: 删除身份证号、手机号等敏感信息
3. **数据加密**: 转换后数据已加密，可安全存储

### 性能考虑

1. **数据量**: 建议 1000-5000 条
2. **向量计算**: 首次转换需要 5-10 分钟
3. **检索速度**: 1000 条数据检索<1 秒
